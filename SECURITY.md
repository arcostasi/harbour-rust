## Reporting

If you believe you have found a security issue, avoid posting exploit details publicly before a maintainer has had a reasonable chance to assess the report.

Preferred path:

1. use the repository's private security reporting feature if it is enabled;
2. otherwise, contact the maintainer through the repository profile or another clearly published private channel;
3. provide a minimal reproduction and impact description.

## Scope

Harbour Rust is alpha software. Reports are especially helpful when they involve:

- unsafe file handling;
- command execution risks;
- malicious source input causing unexpected host-side behavior;
- dependency or packaging issues with real impact.

## Disclosure

The project will prefer coordinated disclosure when practical, but may adapt depending on maintainer availability and project capacity.
