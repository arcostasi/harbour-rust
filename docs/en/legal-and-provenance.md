# Legal and Provenance Guide

## Main Risks

- copying upstream code instead of reimplementing behavior;
- copying documentation wording from books, manuals, or websites;
- using third-party names in a way that implies sponsorship;
- accepting contributions with unclear authorship or rights.

## Preventive Measures

- require original writing for repository documentation;
- use `harbour-core` as an oracle, not as a transplant source;
- keep policy documents explicit about trademark independence;
- ask contributors to confirm originality and submission rights;
- document known provenance-sensitive decisions in pull requests when needed.

## Practical Rule

If a contributor cannot confidently explain where material came from and why it is lawful to reuse, it should not enter the repository.
