# Release 0.5.0-alpha

## Escopo

Esta release consolida o primeiro recorte publicável da Fase 15, mantendo o foco em compatibilidade curada e reversível:

- expansão do corpus PP guiado por `harbour-core/tests/hbpp/`, incluindo `DEFINE CLIPBOARD`, `RELEASE`, `SAVE` e o primeiro corredor `LIST`;
- melhorias estruturais no matcher do PP para restricted markers compostos, como `<prn:TO PRINTER>`;
- renderização por item para result markers blockify sobre listas, cobrindo `<{v}>` em `LIST`;
- manutenção da validação PP com testes unitários, fixtures golden e testes de compatibilidade contra `harbour-core`;
- preservação do escopo alpha: compatibilidade medida por corpus curado, sem promessa de cobertura completa de Harbour.

## Preflight local da release

```text
cargo fmt --all --check
cargo clippy --workspace --all-targets --all-features -- -D warnings
cargo test --workspace
```

## Estado conhecido

- backend principal continua `codegen-c`, sem backend nativo ainda;
- o PP ainda não é uma engine token-based completa como `ppcore.c`;
- `LIST REST`/`LIST ALL` e variações maiores permanecem fora deste recorte de release;
- divergências abertas continuam documentadas em `COMPATIBILITY.md`;
- validação multiplataforma depende dos workflows GitHub Actions para Ubuntu, Windows e macOS.
